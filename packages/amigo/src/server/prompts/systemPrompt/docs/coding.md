---
title: Coding Rules
when: 当任务涉及代码修改、调试、排查当前系统行为，或需要决定是否先设计再实现时
scope: main,sub
---

1. 先确认当前问题属于哪一层：
   - sandbox 仓库代码
   - Amigo app 宿主运行时
   - prompt / tool / orchestration 规则
   - 如果还不确定，先把问题拆成“宿主行为”和“sandbox 行为”两部分分别取证
2. 只要问题和当前系统行为有关，先收集证据再下结论：
   - sandbox 内代码、配置、测试、日志
   - 如果主提示词给了按需文档表，再用 `readRules` 读取相关细则
   - 不要只靠记忆解释当前行为
3. 涉及 UI 或交互改动时，遵循设计优先原则：
   - 先确定设计方向或约束
   - 再做实现
   - 实现后做验证
4. 做出代码修改后，必须运行相应检查；不要只停留在静态推断。
